# Vercel Environment Variables Setup

This document outlines the environment variables required for deploying the RustAcademy frontend to Vercel, with separate configurations for Preview and Production environments.

## Environment Variables

### Required Variables

| Variable Name                                   | Description                    | Preview Value                         | Production Value              |
| ----------------------------------------------- | ------------------------------ | ------------------------------------- | ----------------------------- |
| `NEXT_PUBLIC_ RustAcademy_API_URL`              | Backend API base URL           | `https://api-staging. RustAcademy.to` | `https://api. RustAcademy.to` |
| `NEXT_PUBLIC_SITE_URL`                          | Frontend site URL              | Auto-generated by Vercel              | `https:// RustAcademy.to`     |
| `NEXT_PUBLIC_STELLAR_NETWORK`                   | Stellar network configuration  | `testnet`                             | `mainnet`                     |
| `NEXT_PUBLIC_ERROR_REPORTING_ENABLED`           | Enable client error reporting  | `true`                                | `true`                        |
| `NEXT_PUBLIC_APP_VERSION`                       | Application version            | `1.0.0-preview`                       | `1.0.0`                       |
| `NEXT_PUBLIC_ERROR_REPORTING_URL`               | Error reporting endpoint       | Optional                              | Optional                      |
| `NEXT_PUBLIC_ RustAcademy_ANALYTICS_PUBLIC_KEY` | Analytics public key           | Optional                              | Optional                      |
| ` RustAcademy_INTERNAL_API_URL`                 | Internal API URL (server-side) | Optional                              | Optional                      |

### Network Configuration

#### Preview Environment

- **Stellar Network**: Testnet
- **RPC Endpoint**: `https://horizon-testnet.stellar.org`
- **Horizon Endpoint**: `https://horizon-testnet.stellar.org`

#### Production Environment

- **Stellar Network**: Mainnet
- **RPC Endpoint**: `https://horizon.stellar.org`
- **Horizon Endpoint**: `https://horizon.stellar.org`

## Setting Up in Vercel

### 1. Project Settings

1. Go to your Vercel project dashboard
2. Navigate to **Settings** > **Environment Variables**
3. Add the required variables for each environment

### 2. Preview Environment Variables

For Preview deployments (automatically created for each PR):

```
NEXT_PUBLIC_ RustAcademy_API_URL=https://api-staging. RustAcademy.to
NEXT_PUBLIC_STELLAR_NETWORK=testnet
NEXT_PUBLIC_ERROR_REPORTING_ENABLED=true
NEXT_PUBLIC_APP_VERSION=1.0.0-preview
```

**Note**: `NEXT_PUBLIC_SITE_URL` is automatically set by Vercel for preview deployments.

### 3. Production Environment Variables

For Production deployments (from main branch):

```
NEXT_PUBLIC_ RustAcademy_API_URL=https://api. RustAcademy.to
NEXT_PUBLIC_SITE_URL=https:// RustAcademy.to
NEXT_PUBLIC_STELLAR_NETWORK=mainnet
NEXT_PUBLIC_ERROR_REPORTING_ENABLED=true
NEXT_PUBLIC_APP_VERSION=1.0.0
```

### 4. Environment-Specific Configuration

You can set different values for:

- **Production**: Applied when deploying from the `main` branch
- **Preview**: Applied for all PR preview deployments
- **Development**: Applied for local development

## Mixed Content Prevention

To prevent mixed-content issues with wallet integrations:

1. **Always use HTTPS** for all API endpoints in production
2. **Preview deployments** use HTTPS automatically via Vercel
3. **Wallet connections** require HTTPS in production browsers
4. **CORS headers** are configured in `next.config.ts` to allow proper communication

## Contract Registry URL

The contract registry URL should be configured based on the network:

- **Testnet**: `https://testnet. RustAcademy.to/registry`
- **Mainnet**: `https:// RustAcademy.to/registry`

This is typically derived from `NEXT_PUBLIC_ RustAcademy_API_URL` and `NEXT_PUBLIC_STELLAR_NETWORK` in the application code.

## Validation Checklist

Before deploying, ensure:

- [ ] All required environment variables are set in Vercel
- [ ] Preview environment uses testnet configuration
- [ ] Production environment uses mainnet configuration
- [ ] All URLs use HTTPS (no mixed content)
- [ ] Wallet integration works in both environments
- [ ] Error reporting is configured
- [ ] Analytics is configured (if used)

## Troubleshooting

### Preview Deployment Issues

If preview deployments fail:

1. Check that all environment variables are set for the Preview environment
2. Verify the backend API URL is accessible
3. Check build logs for type/lint errors

### Production Deployment Issues

If production deployments fail:

1. Verify all production environment variables are set
2. Ensure backend API is production-ready
3. Check that wallet integrations work with HTTPS
4. Review build logs for any errors

### Mixed Content Warnings

If you see mixed-content warnings:

1. Ensure all `NEXT_PUBLIC_*` URLs use HTTPS
2. Check that wallet provider URLs use HTTPS
3. Verify no hardcoded HTTP URLs exist in the codebase
